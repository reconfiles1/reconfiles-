What is ReconFiles?

A free browser-based tool for matching records across two CSV files using a shared key — a payment reference, invoice number, or transaction ID. Built for finance teams who need to reconcile payment processor exports (Stripe, GoCardless) against accounting or CRM reports (Salesforce, Xero) without sending sensitive data to a server.

How it works

  1. Upload your two CSV files — drag and drop or click to browse.
  2. Auto-detect scans the actual values in each file and suggests the best matching columns, even when names differ between systems.
  3. Review the suggested key and amount columns, adjust if needed, set your tolerance, then click Reconcile.
  4. Export a multi-sheet Excel workbook — matched records, unmatched rows from each file, and any amount discrepancies, with VLOOKUP formulas linking both source sheets.

Features

  • Smart column detection — finds the right key even when it's called "Description" in one file and "Asperato Reference" in another.
  • FX tolerance — absolute (£) and percentage (%) thresholds so small currency conversion differences don't create false mismatches.
  • Excel export — multi-sheet workbook with live VLOOKUP formulas, frozen headers, and auto-sized columns.
  • Saved mappings — save your column configuration and reload it next month in one click.
  • Completely private — all processing runs in your browser. Your files are never uploaded anywhere.
  • Free, no account needed — open the page and start reconciling.
